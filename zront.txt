### PROMPT 2 — Database Schema
```
Create the following real tables in our cloud 
backend. No dummy fields, no placeholder columns.

Table: customers
  - id: uuid, primary key
  - first_name: varchar(100), not null
  - last_name: varchar(100), not null
  - email: varchar(255), unique, not null
  - phone: varchar(20)
  - created_at: timestamp, default now()
  - updated_at: timestamp

Table: addresses
  - id: uuid, primary key
  - customer_id: uuid, foreign key → customers.id
  - address_line1: varchar(255), not null
  - address_line2: varchar(255)
  - city: varchar(100), not null
  - state: varchar(100), not null
  - country: varchar(100), not null, default 'Nigeria'
  - is_default: boolean, default false
  - created_at: timestamp, default now()

Table: products
  - id: uuid, primary key
  - name: varchar(255), not null
  - slug: varchar(255), unique, not null
  - description: text
  - category: enum('hair_care','men','women',
    'bags','shoes'), not null
  - price: numeric(10,2), not null
  - compare_at_price: numeric(10,2)
  - stock_quantity: integer, not null, default 0
  - is_available: boolean, default true
  - images: jsonb
  - benefits: jsonb
  - created_at: timestamp, default now()
  - updated_at: timestamp

Table: orders
  - id: uuid, primary key
  - order_number: varchar(20), unique, not null
  - customer_id: uuid, foreign key → customers.id
  - shipping_address_id: uuid, 
    foreign key → addresses.id
  - status: enum('pending','confirmed',
    'processing','shipped','delivered',
    'cancelled'), default 'pending'
  - subtotal: numeric(10,2), not null
  - shipping_fee: numeric(10,2), not null
  - total: numeric(10,2), not null
  - notes: text
  - created_at: timestamp, default now()
  - updated_at: timestamp

Table: order_items
  - id: uuid, primary key
  - order_id: uuid, foreign key → orders.id
  - product_id: uuid, foreign key → products.id
  - product_name: varchar(255), not null
  - product_price: numeric(10,2), not null
  - quantity: integer, not null
  - subtotal: numeric(10,2), not null
  - created_at: timestamp, default now()

Table: order_status_history
  - id: uuid, primary key
  - order_id: uuid, foreign key → orders.id
  - status: enum (same as orders.status)
  - note: text
  - changed_at: timestamp, default now()
```

### PROMPT 3 — Cart
```
Implement a persistent cart using React context 
and localStorage. No cart table in the database.

CART STATE:
- Lives in a CartContext wrapping the entire app
- Each cart item holds:
  product_id, name, price, quantity, image
- Cart persists across page refreshes 
  via localStorage
- Cart is cleared on successful order submission

CART ACTIONS:
- addToCart(product): 
  if product already in cart, increment quantity
  if new product, add with quantity 1
- removeFromCart(product_id):
  removes item completely
- updateQuantity(product_id, quantity):
  if quantity reaches 0, remove item
- clearCart():
  empties cart, clears localStorage

NAVBAR CART ICON:
- Thin line cart icon, right side of navbar
- Badge showing total item count:
  background #C4A882, color #1A1A1A,

- Clicking icon opens cart drawer

CART DRAWER:
```

### PROMPT 5 — API & Order Submission
```
Connect checkout form to the cloud backend
via these API calls:

POST /api/orders
  Request body:
  {
    customer: {
      first_name, last_name, email, phone
    },
    shipping_address: {
      address_line1, address_line2,
      city, state, country
    },
    items: [
      { product_id, quantity }
    ],
    notes: string (optional)
  }
  Response:
  {
    order_id,
    order_number,
    status,
    total,
    created_at
  }

GET /api/orders/:order_number
  Returns full order details including
  items and full status history

ERROR HANDLING:
- If API call fails show an inline error 
  above the Confirm Order button:
  Inter 12px, color #C0392B
  "Something went wrong. Please try again."
- Do not clear the form on failure
- Disable the Confirm Order button while 
  the request is in flight
- Show a subtle loading state on the button:
  text changes to "Placing Order..."
  opacity 0.7, cursor not-allowed

STOCK:
- On successful order submission decrement 
  stock_quantity in the products table 
  for each ordered item by its quantity
- If any item in the cart has 
  stock_quantity === 0 before submission:
  block checkout, show inline message:
  "[Product Name] is no longer available"
  Inter 12px, color #C0392B
```
